### Reflection
This task was alot more fun as I got to understand how brightness/alpha, rgb are processed as values. 
They are all 8 bit channels and the way the glitch distorting effect works is that it sorts the pixels from darkest to brightest, creating a distorted effect. It doesn' actually calculate based on the actual brightness of the pixel but its numerical value, sorting it from smallest to biggest and then displaying it by each row. 
With the code given from the blg post about glitch art, I decided to modify it and experiment with it to see the effects I can acheive. 

I have then created another version where the pixels are sorted from the other way round ( in an inverted way). SO rather the dark pixels fading to bright, the bright pixels fade to dark creating a beautiful silouette
